3.8. Управление пользователями (административный модуль)

3.8.1. Общее описание

Административный модуль управления пользователями доступен исключительно пользователям с ролью ADMIN. Доступ к странице /admin/users защищён на уровне серверного компонента: при отсутствии сессии или несоответствии роли вызывается функция notFound(), возвращающая ответ 404 без раскрытия информации о существовании страницы.

Модуль предоставляет три операции: создание новой учётной записи, изменение роли существующего пользователя и активация/деактивация учётной записи.

3.8.2. Создание пользователей

Создание учётной записи выполняется серверным действием createUser() (модуль server/admin/user-actions.ts). Входные данные проходят валидацию Zod-схемой. Правила валидации приведены в таблице 10.

Перед созданием учётной записи проверяется уникальность email-адреса: при наличии существующего пользователя с таким адресом возвращается ошибка без создания записи.

Пароль хэшируется функцией bcrypt.hash(password, 12) с коэффициентом сложности 12 раундов. Хэш сохраняется в поле passwordHash; открытый текст пароля нигде не сохраняется. Новая учётная запись создаётся с флагом isActive = true.

3.8.3. Изменение роли и деактивация

Изменение роли и статуса активности выполняется через API-эндпоинт PATCH /api/users/{id}. Операции доступны только администратору. Предусмотрена защита от самодеактивации и самоизменения роли: администратор не может изменить собственную роль или деактивировать собственную учётную запись — соответствующие элементы управления на странице недоступны при совпадении идентификатора пользователя с идентификатором текущей сессии.

Деактивация учётной записи (isActive = false) немедленно блокирует вход пользователя в систему: функция authorize() в конфигурации NextAuth проверяет флаг isActive до сравнения пароля и отклоняет аутентификацию для деактивированных учётных записей. Физическое удаление учётных записей не предусмотрено — используется только деактивация, что сохраняет ссылочную целостность данных и историю аудита.

3.8.4. Отображение списка пользователей

Страница отображает таблицу всех пользователей системы с полями: ФИО, email, роль, статус активности и кнопки управления. Список сортируется по роли, затем по имени. Роль пользователя изменяется непосредственно через выпадающий список в строке таблицы без перезагрузки страницы (клиентский PATCH-запрос). Статус активности отображается цветовой индикацией: зелёный — активен, красный — деактивирован.
