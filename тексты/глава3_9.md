3.9. Стадии и этапы реализации

Разработка системы защиты электронного документооборота выполнялась последовательно по этапам, каждый из которых завершался работоспособным результатом, на который опирался следующий этап. В таблице 11 приведены этапы реализации системы.

Таблица 11 – Этапы реализации системы

[ВСТАВИТЬ ТАБЛИЦУ ИЗ EXCEL — лист «3.9 Этапы реализации»]

Этап 2 (Проектирование базы данных) стал фундаментом всей системы. На этом этапе были приняты архитектурные решения, определившие дальнейшую реализацию: использование soft delete вместо физического удаления, каскадные операции для связанных сущностей, стратегия Restrict для подписей (запрет удаления подписанных документов), SetNull для журнала аудита (сохранение истории при удалении пользователей).

Этап 5 (Ролевая модель) реализовывался параллельно с этапом документооборота. Политики доступа намеренно вынесены в отдельный модуль (server/policies/) в виде чистых функций без побочных эффектов, что упрощает их тестирование и повторное использование во всех серверных обработчиках.

Этап 7 (Электронная цифровая подпись) потребовал наибольшей технической проработки. Формат хранения файла подписи (четыре строки: сертификат, хэш атрибутов, хэш контента, PKCS#7) был выбран для обеспечения независимой проверки целостности содержимого и криптографической корректности подписи.

Этап 9 (Дополнительные средства защиты) реализовывался на протяжении всей разработки по мере появления соответствующих функций: валидация добавлялась при реализации каждого серверного действия, защита файловых операций — при реализации загрузки вложений, rate limiting — при реализации аутентификации.
